## §0. 도입 (슬라이드 1–3)

### 슬라이드 1 — 표지
- 제목: **"Aria Glass 실시간 3DGS에서 Floater 제거하기 — 3일간의 스프린트"**
- 부제: 손실 설계 → 지표 재발명 → 교차 장면 일반화 → 라벨 없는 전자동 파이프라인

### 슬라이드 2 — 배경: 무엇을 만들고 있나
- 파이프라인 전체 그림: **Aria Glass(스마트글래스) 카메라 스트림 → OpenMAVIS(ORB-SLAM) → 카메라 pose + 희소 지도 → 3D Gaussian Splatting 학습 → 방의 3D 복원**
- 목표: **분 단위 turnaround**로 geometry 좋은 실내 3D 복원 (실시간 경로라 MPS 후처리 불가 → ORB 트랙이 본선)
- 실험 무대: 301동 1253호(주 무대) + 이후 305호, 12층 로비, 회전 궤적 데이터
- [시각자료] 파이프라인 블록 다이어그램 (VRS → EuRoC → SLAM → 3DGS), Aria 기기 사진, 1253호 복원 전경
- [발표 노트] "3대 벤치마크 = 학습시간 / PSNR / floater 개수"를 여기서 선언

### 슬라이드 3 — 문제: Floater란 무엇인가
- 정의: 학습 결과에 생기는 **허공에 떠 있는 반투명/불투명 gaussian 찌꺼기** — 시점을 움직이면 유령처럼 떠다녀 체감 품질을 크게 해침
- 두 집단 프레임(기존 발견): Pop1 = SLAM init outlier(해결됨) / **Pop2 = densification이 만들어내는 floater(이번 스프린트의 표적)**
- 출발점 자산: 사용자가 SuperSplat으로 직접 지운 **수동 라벨 2,817개** (1253호)
- [시각자료] floater 확대 렌더 3컷(허공 먼지/가시 blob), SuperSplat 라벨링 화면 캡처
- [발표 노트] "라벨이 있다는 것"이 이 스프린트 전체를 가능하게 한 핵심 자산

## §1. 토요일(07-11): 기존 방법의 사망 선고와 새 손실의 설계 (슬라이드 4–7)

### 슬라이드 4 — 기존 접근(Plateau Loss)이 왜 안 되는지 학습 없이 증명
- 질문: "내가 만든 plateau loss로 저 2,817개를 지울 수 있나?" → **분석만으로 불가능 판정**
- 4중 구조적 실패: ① floater의 66%가 plateau '안'에 있어 gradient=0 ② 판별력 AUC 0.511(무작위 수준) ③ 적응형 tau가 판별력을 파괴(raw 거리는 AUC 0.93인데!) ④ λ 크기는 애초에 문제가 아니었음
- 교훈: **신호는 존재한다. 손실이 그 신호를 버리고 있었을 뿐.**
- [시각자료] plateau 개념도(구형 평탄 영역) + "floater가 plateau 안에 있는" 단면도, AUC 막대(0.511 vs 0.93)
- [발표 노트] "학습 없이 분석만으로 기각" — 이 스프린트의 방법론적 특징(빠른 가설 기각)을 강조

### 슬라이드 5 — Carve Loss의 핵심 아이디어: 빈 공간의 증거
- 직관: **카메라가 SLAM 점을 봤다면, 그 사이 공간은 비어 있어야 한다** (ray casting → free-space carving)
- score 구성: voxel마다 transit(통과 증거)/terminal(표면 증거) 집계 → ρ = transit/(transit+3·terminal) → w(x) = ρ · min(d5_SLAM/0.25, 1) → **champion score = w·(1−maxop_5cm)**
- 수동 라벨 판별력: **AUC 0.974~0.98** (plateau 0.511 대비)
- 추가 입력 불필요: SLAM 점 + 카메라 pose만으로 계산 (학습 중 0.4초/갱신)
- [시각자료] ray casting 개념도(카메라→점, 통과 voxel 초록/표면 voxel 빨강), score 히트맵 단면, AUC 비교 막대
- [발표 노트] 여기가 기술 핵심 1페이지 — 시간 들여 설명

### 슬라이드 6 — 손실로 만들기: 4개 구성요소
- ① **soft**: score 높은 점에 opacity 압력 (λ0.02 — 자가 교정적: 진짜 표면은 RGB gradient가 저항)
- ② **prune**: 시각 기여 예산 상한부 top-K 삭제 (총 0.75% 상한 — 표면 손실 통제)
- ③ **birth gate**: densify 신생아 중 score>0.95 즉시 삭제 (분석상 허공 출생 91% 차단; 실제로 split의 29.5%가 허공 출생)
- ④ **force**: carve-potential φ(빈 공간→표면 거리장)의 -∇ 방향 xyz 견인 (3D force의 부활)
- 부수 피해 통제 설계: 원안 prune은 표면 기여 3.83% 손실이라 폐기, 안전 규칙은 0.39% 손실/구멍 0
- [시각자료] 4요소 아이콘 다이어그램 + 각 요소가 작동하는 위치(출생/성장/정리) 타임라인
- [발표 노트] "덮어놓고 지우면 표면에 구멍" — 부수 피해 정량화가 설계의 절반

### 슬라이드 7 — 토요일 밤~일요일 새벽: 7연속 학습으로 레시피 확정 (exp38–40)
- 학습이 회당 ~10분임을 확인하고 하룻밤 7 run 강행
- 결과: **exp40b(softlite+force) 채택** — PSNR 무손실로 region 먼지 -86%, 가시 먼지 -45%(force 무비용 기여)
- gradient 프로브: floater는 '진동 평형' 위에 앉아 있음 → force가 일관 편향을 주면 이탈
- [시각자료] 7 run 결과표(PSNR/먼지), exp30r vs exp40b 렌더 A/B(사용자 확인 완료 — "진짜 없어진 게 체감된다")
- [발표 노트] 사용자 육안 검수 코멘트 인용 가능

## §2. 일요일(07-12) 오전: 지표의 재발명 (슬라이드 8–10)

### 슬라이드 8 — 충격 발견: Floater는 train PSNR 기생충이다
- 렌더 A/B 중 발견: **수동으로 floater를 지우기만 해도 train PSNR이 -3.7dB 떨어진다**
- 의미: floater는 학습 이미지의 잔차를 흡수하며 PSNR을 '벌어주는' 존재 → **train PSNR로 floater 방법을 평가하면 안 됨** (지표 금지 선언)
- 이 발견이 이후 모든 실험의 평가 체계를 바꿈
- [시각자료] "지우기만 했는데 -3.7dB" 개념 만화(먼지가 잔차를 가리는 그림), 금지 표시 붙은 train PSNR
- [발표 노트] 이 스프린트에서 가장 중요한 단일 발견. 시간 들여 설명

### 슬라이드 9 — 새 평가 체계 3종 세트
- ① **Region GT 지표**: 라벨 floater들의 위치로 3D 삭제 영역을 사면체 채움(tetra)으로 구성(85.4m³) → 어떤 학습 결과든 "그 영역 안 점 수/가시 점 수"로 채점 (라벨 재사용 가능한 GT)
- ② **held-out 세그먼트 분할**: novel-view 일반화 측정 (train PSNR 대체)
- ③ **SSIM/LPIPS**: 지각 품질 — exp41/42로 "softlite -0.44dB는 비지각적(SSIM/LPIPS 무차이)" 확인 = 먼지 제거 비용의 Pareto floor
- 부산물: 규칙 기반 floater 추출기(예산 top-K, safe/balanced/aggressive 프리셋)
- [시각자료] tetra region 3D 시각화 스크린샷, 지표 3종 요약표
- [발표 노트] "지표를 다시 만든 뒤에야 공정한 실험이 가능해졌다"

### 슬라이드 10 — 목표 재정의와 전장 확대
- 프로젝트 목표 명문화: **실시간 스트림 → 분 단위 → 좋은 geometry** (MPS 불가 → ORB 본선)
- 이를 위한 3대 벤치마크: **학습시간 / PSNR / floater 수**
- 학습 속도를 위한 선행연구 조사: EDGS(dense correspondence init), Instant-GI(2D GS→3D lift), depth-mono — "좋은 init이 곧 속도이자 품질"
- [시각자료] 3대 벤치마크 다트보드, EDGS/Instant-GI 논문 썸네일
- [발표 노트] 여기서부터 "init 전쟁" 시작을 예고

## §3. 일요일 오후~밤: Init 전쟁 (슬라이드 11–13)

### 슬라이드 11 — Init 엔지니어링 4원칙 (exp44 트랙)
- ① **먼지는 init에서 잡는다**: dense init을 carve score로 사전 필터(init 점의 70%가 빈 공간에 있었음 → -96%)
- ② **색은 선불**: init 점에 투영 색 입히기만으로 +1.0~1.6dB
- ③ **갭은 배치**: snap-to-surface로 표면 빈틈 메움
- ④ **용량은 densify 3k로 충분**: no-densify는 과소용량
- 실전 적용: Instant-GI 사전학습 가중치 다운로드·추론(학습된 위치확률맵), EDGS식 RoMA dense correspondence 삼각측량(재투영 <2px 검증)
- [시각자료] init 포인트클라우드 진화 4단계(raw SLAM → dense → 필터 → 착색), RoMA 매칭 시각화
- [발표 노트] 각 원칙마다 정량 근거 한 줄씩

## §4. 일요일 밤~월요일 새벽: 교차 장면 검증 — 실패, 진단, 처방 (슬라이드 14–18)

### 슬라이드 14 — 질문: 다른 방에서도 되는가? (exp43 설계)
- 사용자가 3개 장면 직접 라벨링: **1253_rot(같은 방·회전 궤적) 5,168개 / 305호(다른 방) 9,059개 / 12F 로비 1,449개** (복도 2곳은 기하 열화로 라벨 불가 → 제외)
- 사전 등록 기준: score AUC ≥0.95, carve 학습으로 가시 먼지 ≥90% 감소
- [시각자료] 3장면 썸네일 + 라벨 통계표, 사전등록 기준 박스
- [발표 노트] "판정 기준을 실험 전에 못 박았다"

### 슬라이드 15 — 결과 ①: 같은 방은 완벽, 다른 방은 실패
- **1253_rot: 대성공** — 자동 pseudo-label 정밀도 100%/재현율 90.2%, score AUC 0.9813 → **같은 방은 재라벨링 없이 전자동**
- **305: 첫 실패** — AUC 0.7993, pseudo-label 정밀도 44%
- **12F도 동일 패턴** — AUC 0.858
- ORB confidence(observations 등): 전 장면 무효과 → 최종 기각
- [시각자료] 3장면 AUC 막대(0.98/0.80/0.86), rot pseudo-label 벤 다이어그램(100% 정밀도)
- [발표 노트] "실패가 났다 — 이제 원인 규명 파트"

### 슬라이드 16 — 진단: 원인은 'ORB 지도의 표면 커버리지'
- 배제 과정: tau/voxel 스케일 스윕(0.77~0.78, 무효) → 라벨 성격 분해(탐지가능 모집단에서도 0.82, 무효) → **정상 표면점조차 SLAM 점에서 중앙값 0.3m 떨어져 있음** = "SLAM 근접=표면" 가정 붕괴
- 부수 발견: 305 라벨의 ~30%는 free-space 부유물이 아닌 **표면 부착 blob** (다른 문제 클래스로 분리)
- [시각자료] d5 분포 히스토그램(라벨 vs 생존, 1253 vs 305 대비), 배제 과정 플로차트
- [발표 노트] 체계적 배제 과정 자체가 보여줄 가치

### 슬라이드 17 — 처방: Depth-Anchor Carve (이 스프린트의 대표 성과)
- depth-pro(단안 depth, 장면당 33초)로 **조밀 표면 앵커** 생성 → SLAM 점 대신 carve field 재구성
- 점수 회복: 305 AUC **0.80 → 0.905** (+이미지 공간 depth-violation 신호 결합)
- **학습 재현 성공**: 305 먼지 **-83%**, 가시 **-76%**, PSNR 동급 — **재현 런까지 정밀 일치(173→179)로 확정**
- [시각자료] SLAM 앵커(6k, 듬성) vs depth 앵커(573k, 조밀) 포인트클라우드 비교, 305 before/after 렌더, 재현성 표
- [발표 노트] "앵커 커버리지만 고치면 다른 방에서도 재현된다" — 교차 장면 관문 통과 선언

### 슬라이드 18 — 과학적 반전: 대조군이 잡은 유령
- 미스터리: rot에서 carve 학습마다 가시 먼지가 106→700~1,300으로 '역증가'하는 듯 보임
- 가설 3연속 기각: 응집 도피(부검: 먼지 65%가 새 위치) → force 항(끄고도 812) → 질량 재분배(증폭이었음)
- **대조군의 심판: baseline 재실행 = 가시 1,091** → 원래 106이 행운의 런. **carve 무죄, 진범은 run-to-run 분산(×10)**
- 교훈 두 개: ① **먼지 지표도 단일 런 비교 금지**(재현 런 필수) ② 회전 궤적 장면은 지표 자체가 불안정한 클래스
- [시각자료] "용의자 보드" (응집/force/재분배/분산 — 앞 3개 X, 분산 O), baseline 2회 비교 막대
- [발표 노트] 발표의 백미 — 실패한 가설들을 숨기지 않고 추적 과정으로 보여주기

## §5. 월요일(07-13): 전자동 파이프라인 완성 (슬라이드 19–21)

### 슬라이드 19 — 라벨 없는 앵커 자가진단 (마지막 퍼즐)
- 새 장면마다 "SLAM 앵커 vs depth 앵커 중 뭘 믿나"를 라벨 없이 판정하는 **2규칙**:
  - 규칙1: SLAM 자기 NN p50 < 0.05m → **SLAM 앵커** (1253 0.042 / rot 0.043 — AUC 0.97+)
  - 규칙2: depth 교차프레임 불일치 p50 < 0.04 → **depth 앵커** (305: 0.032 — AUC 0.886)
  - 규칙3: 둘 다 실패 → **문제 클래스 경고 + carve OFF** (12F: 실제로 carve가 -1dB 손해였음을 학습으로 확인)
- 4/4 장면에서 실측 최선 앵커와 일치
- [시각자료] 진단 플로차트(3분기), 4장면 지표×AUC 매트릭스
- [발표 노트] "이 진단기 덕에 새 장면 파이프라인이 사람 개입 없이 닫혔다"

### 슬라이드 20 — 보너스: SLAM 포인트 없이도 floater를 잡는다
- 질문(사용자): "SLAM 포인트가 아예 없으면?"
- 답: depth-pro 절대 깊이(공장 캘리브레이션으로 focal 검증) + **pose-기하 스케일 자가 보정**(스테레오+IMU 덕에 pose가 미터 단위 → 교차 프레임 일관성으로 스케일 유일 결정)
- 12F 성적: raw 0.855 → 자가 보정 **0.893** (SLAM 보정 상한 0.908의 98%) — **지도 포인트 0개로 성립**
- 단, 학습 통합(vr 채널)은 무효과 → **"탐지≠제거" 간극**: underfit 장면에선 이미지가 먼지를 요구해 어떤 압력도 RGB gradient를 못 이김 → vr의 용도는 오프라인 청소·pseudo-label·탐지
- [시각자료] pose-기하 자가 보정 개념도(두 카메라+depth 삼각형), AUC 사다리(0.855→0.893→0.908)
- [발표 노트] floater의 근본 해법은 압력이 아니라 재구성 품질이라는 통찰로 연결

### 슬라이드 21 — 최종 산출물: 진단-분기 전자동 파이프라인
```
새 장면 VRS
  → SLAM (OpenMAVIS)
  → [자가진단 2규칙]
       ├─ SLAM 양호   → SLAM-앵커 carve      (1253·rot에서 검증)
       ├─ SLAM 희소    → depth-앵커 carve     (305에서 재현 확정, depth 33초)
       └─ 둘 다 불량   → carve OFF + baseline  (12F에서 -1dB 증거)
  → (같은 방 재방문 시) pseudo-label 자동 청소 (정밀도 100%)
```
- 3대 벤치마크 최종: 품질 **+0.9dB** / 먼지 **-83~88%** / 속도 **8분**(fast) — 사람 라벨링 0회로 새 장면 적용 가능
- [시각자료] 위 파이프라인 다이어그램(발표의 마지막 그림), 벤치마크 3종 최종 스코어카드
- [발표 노트] "토요일엔 손실 하나였고, 월요일엔 파이프라인이 됐다"

## §6. 마무리 (슬라이드 22)

### 슬라이드 22 — 결론과 남은 과제
- 확정된 것 5줄:
  1. Carve Loss: 빈 공간 증거 기반 score(AUC 0.98) + 4요소 손실 — PSNR 무손실 먼지 -86%
  2. train PSNR은 floater 지표로 무효(기생충) → region GT + held-out + SSIM/LPIPS 체계
  3. 교차 장면 재현: 앵커 품질이 단일 지배 변수 — depth-anchor 처방으로 305 재현(재현 런 포함)
  4. 라벨 없는 자가진단 2규칙 + 진단-분기 파이프라인 (4/4 장면)
  5. 연구 위생: 먼지 지표 분산 ×10 → 재현 런 의무화
- 남은 과제:
  - 회전 궤적/underfit 장면: 탐지≠제거 간극 — init·시차 개선 축(압력으론 불가)
  - 표면 부착 blob (305 라벨의 30%): free-space 신호 밖의 새 문제 클래스
  - 파이프라인 원커맨드화 + 새 장면 1개로 자가진단 5번째 검증
- [시각자료] 확정 5 + 과제 3 요약 카드
- [발표 노트] 질문 유도: "여러분이라면 탐지≠제거 간극을 어떻게 뚫겠습니까"
