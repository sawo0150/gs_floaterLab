## §1. 문제의식과 기존 접근 (슬라이드 4–6)

### 슬라이드 4 — 문제의식: 왜 3D 손실이 필요하다고 생각했나 (+ Plateau Loss 소개)
- **Photometric loss의 구조적 사각지대**:
  - photometric loss는 **렌더된 픽셀에 기여하는 gaussian에만 gradient를 전달** → 카메라에서 안 보이거나 드물게 보이는 gaussian은 신호 자체가 없음
  - ray 분포가 **Z 높이별·위치별로 극단적으로 불균등** (많이 훑는 곳 vs 거의 안 훑는 곳)
  - depth loss도 3차원 정보를 주긴 하지만 **역시 ray 기반** — 같은 사각지대 공유
  - → **3차원 공간에 균등하게 신호를 주는 손실이 있어야 기하를 골고루 잡을 수 있다**는 가설
- **최적화 관점의 문제의식**:
  - floater는 **local minima에 갇히는 경향** + 일반 신경망 파라미터와 달리 **gaussian 하나의 차원은 매우 작음** → 저차원에서는 saddle point보다 **local minima가 지배적** → photometric gradient만으로는 탈출이 어려움
  - → **loss landscape를 내려가기 쉽게 smoothing할 필요**
- **⇒ 결론: 3D로 손실을 구현해 (1) 공간 균등 gradient, (2) smoothing으로 local minima 탈출 경로를 만들자**
- **Plateau Loss = 이 문제의식의 1차 구현**:
  - 각 gaussian에 대해 가장 가까운 anchor(SLAM/depth 표면점)까지의 정규화 거리 D를 정의
  - 표면 근처 반경 tau 안쪽은 **평탄(plateau, gradient 0)** — 표면 점을 건드리지 않음
  - plateau 바깥은 D에 비례한 페널티 → 허공 점을 표면 쪽으로 견인
  - 변형들: 구형/타원형(ellipsoidal) plateau, 적응형 tau(anchor 밀도 따라 확대), DepthPro anchor 결합
- [시각자료] ① ray 커버리지 히트맵(높이별 ray 밀도 불균등 — `check_plateau_vs_raydensity.py` 산출물 재활용) ② 1D loss landscape 만화: 뾰족한 local minima vs smoothing 후 ③ plateau 함수 모양 그래프(평탄 구간 + 선형 증가)
- [발표 노트] "이 문제의식 자체는 지금도 유효하다 — 문제는 구현이었다"로 다음 슬라이드 예고

### 슬라이드 6 — Plateau의 사망 선고: 학습 없이 분석만으로 기각
- 질문: "plateau loss로 저 2,817개를 지울 수 있나?" → **분석만으로 불가능 판정** (`verify_plateau_capability.py`)

## §2. Carve Loss: 설계 (슬라이드 7–8)

### 슬라이드 7 — Carve Loss의 핵심 아이디어: 빈 공간의 증거
- 직관: **카메라가 SLAM 점을 봤다면, 그 사이 공간은 비어 있어야 한다** (ray casting → free-space carving) — ray가 안 닿는 곳이 아니라 **ray가 '통과했다'는 적극적 증거**를 쓰므로 photometric 사각지대와 무관하게 3D 전역에 정의됨 (슬라이드 4 문제의식의 2차 구현)
- Field 구성 (학습 시작 시 1회, CPU):
  - 카메라→SLAM점 ray를 6cm 간격으로 행진시키며 voxel(10cm)에 **transit(통과) 증거** 적립, 종점 ±20cm에 **terminal(표면) 증거** 적립
  - ρ(x) = transit / (transit + 3·terminal) — "이 voxel이 빈 공간일 증거비" (3×3×3 smoothing — 문제의식의 'smoothing' 요구 반영)
- Score 구성: **w(x) = ρ(x) · min(d5_SLAM/0.25m, 1)** (증거비 × anchor 거리) → **champion score sᵢ = w(xᵢ)·(1−maxop_5cm)** (주변 5cm에 불투명 이웃이 있으면 보호)
- 수동 라벨 판별력: **AUC 0.974~0.98** (plateau 0.511 대비) / 학습 중 갱신 0.38초, 추가 입력 불필요(SLAM+카메라)
- [시각자료] ray casting 개념도(카메라→점, 통과 voxel 초록/표면 voxel 빨강), score 히트맵 단면(방 수평 슬라이스), AUC 비교 막대
- [발표 노트] 기술 핵심 1페이지 — 시간 들여 설명. "plateau가 버리던 raw 거리 신호(0.93)를 증거비와 결합해 0.98로"

### 슬라이드 8 — 손실로 만들기: 4개 구성요소 (수식 상세)
- **① Soft opacity 압력** — score가 높은(빈 공간에 있는) 점의 opacity를 지그시 누름:
  ```
  L_soft = λ_soft · mean_{i: sᵢ > s_min} ( sᵢ · σ(oᵢ) )        [sᵢ는 detach — gradient는 opacity로만]
  champion: λ_soft = 0.02, s_min = 0.5 (softlite)
  ```
  자가 교정적: 진짜 표면이 잘못 눌리면 RGB gradient가 저항해 되살림. λ0.05는 -0.3dB 추가 비용 → 0.02가 스위트스팟
- **② 예산 상한부 Budget Prune** — 시각 기여량으로 부수 피해를 회계 관리하며 삭제:
  ```
  contribᵢ = oᵢ · (s₂·s₃)ᵢ · visᵢ            [불투명도 × 단면적(중간·최대축) × 관측수]
  score 내림차순 정렬 → 누적 contrib ≤ budget 까지 top-K 삭제 (단 sᵢ > 0.3만)
  champion: cycle당 0.05%, 총 상한 0.75% (1k iter마다, 7k 이후)
  ```
  원안(무예산)은 표면 기여 3.83% 손실로 폐기 — 예산제로 0.39% 손실/구멍 0
- **③ Birth Gate** — densify 직후 신생아 검문:
  ```
  kill = (birth_step == 지금 iter) & (sᵢ > 0.95)     [허공 출생 91% 차단 — split의 29.5%가 허공 출생]
  ```
- **④ Carve-potential Force** — 3D 위치를 직접 견인하는 항 (문제의식의 '3D 균등 gradient' 직접 구현):
  ```
  attractor = {terminal 증거 > 1 voxel} ∪ {가시 gaussian(op>0.3) 점유 voxel}
  φ(x) = EDT(¬attractor) · 0.1m                      [빈 공간→가장 가까운 표면까지 거리장]
  L_force = λ_force · mean_{i: sᵢ > 0.5} ( sᵢ · φ(xᵢ) )   [φ는 trilinear 보간 — gradient가 xyz로]
  champion: λ_force = 0.05
  ```
  gradient 프로브 근거: floater는 강한 핀이 아니라 **진동 평형**(gradient coherence 0.257) 위에 앉아 있음 → 약한 '일관' 편향이면 이탈 가능
- 4요소의 분업: gate(출생 차단) → soft(성장 억제) → force(위치 견인) → prune(정리). 전부 같은 score 하나에서 파생
- [시각자료] 각 수식 박스 4개 + 타임라인(iter 0—1k—7k—30k 위에 각 요소 활성 구간), φ 거리장 단면 히트맵
- [발표 노트] "누르고(1), 자르고(2), 안 낳게 하고(3), 끌어당긴다(4)" 한 줄 요약. detach 위치(score는 상수 취급) 질문 대비

- 성분 분해 판독: **soft가 가시 먼지의 주역**(38b vs 38a/c에서 확인), **force는 무비용 추가 억제**(38b→40a: region -25%·가시 -45%), 결합(40b)이 각각보다 우수 = 시너지
- [시각자료] 위 표 + 렌더 A/B(exp30r vs exp40b — 사용자 육안 검수: "진짜 없어진 게 체감된다"), 성분 기여 워터폴 차트(baseline→+gate/prune→+force→+softlite 단계별 먼지 감소)
- [발표 노트] 표의 재미 포인트: 38a는 먼지 최저급이지만 PSNR -0.3dB(과압), N 13만으로 수축 — "세게 누르면 빨래도 준다"

### 슬라이드 11 — 재현성·전이·메커니즘 증거
- **챔피언 재현(exp40br)**: PSNR 32.448(Δ0.13 — 노이즈 내), region 462/가시 25 (1차 498/28) → **먼지 억제 고재현성**. 정직한 판독: 챔피언 PSNR 범위는 baseline 하단 접경 — 실비용 0~0.2dB
- **MPS 트랙 전이(exp39b, softlite+force)**: PSNR **-0.10dB**에 **가시 먼지 96→0개**, 먼지 기여 6.42→**0.21%** → 양 트랙 공통 채택
- **메커니즘 증거 2종** (계측 인프라: `split_events.pkl` 출생 로그 + `accum_rgb_grad_vec`):

- [시각자료] 재현 run 비교 막대(1차 vs 재현), MPS before/after 렌더(가시 먼지 96→0), 허공 출생 파이차트(29.5%), coherence 히스토그램

## §4. 지표의 재정비 (슬라이드 12–13)

### 슬라이드 12 — 평가 체계 재구축: train PSNR의 사각지대와 3종 지표
- 렌더 A/B 중 확인: **수동으로 floater를 지우기만 해도 train PSNR -3.7dB** — floater가 train 이미지의 잔차(글레어·노출)를 흡수해 PSNR을 벌어줌 → train PSNR은 floater 평가에 부적합 (슬라이드 6의 "RGB gradient 2.23×"의 귀결)
- 재구축한 3종 체계:
  - ① **Region GT 지표** (슬라이드 5의 사면체 영역 재사용): 어떤 학습 결과든 "영역 안 점/가시 점 수"로 채점
  - ② **held-out 평가**: every-8th는 보간이라 기생충이 test에서도 이김(-0.46dB) → **연속 세그먼트 hold-out**으로 교체, 두 split 일치로 비용 확정
  - ③ **SSIM/LPIPS**: 챔피언 vs baseline **사실상 동률**(SSIM 0.9152 vs 0.9167, LPIPS 0.3668 vs 0.3633)
- **Pareto 지도(exp42)** — 먼지↔PSNR 트레이드는 연속 조절 가능:

| 운영점 | test PSNR Δ | region_n | 가시 |
|---|---:|---:|---:|
| baseline | — | ~3,700 | ~200 |
| prune+gate | -0.09 | 1,795 | 217 |
| +force | -0.21 | 1,136 | 149 |
| softlite+force (챔피언) | **-0.44** | **~460** | **~25** |

- 최종 그림: **챔피언 = 지각 품질 동등 + 가시 먼지 -90%+ + PSNR -0.44dB(비지각적 잔차 맞춤의 상실)** — λ·budget으로 운영점 조절 가능
- [시각자료] Pareto 곡선(x=먼지, y=test PSNR, 운영점 4개 표시), SSIM/LPIPS 동률 표, region 영역 3D 오버레이(슬라이드 5 재등장)
- [발표 노트] 담백하게: "지표를 고쳐야 공정한 실험이 가능했다" — 과장 없이 체계 소개 중심

### 슬라이드 13 — 목표 재정의와 전장 확대
- 프로젝트 목표 명문화: **실시간 스트림 → 분 단위 → 좋은 geometry** (MPS 불가 → ORB 본선)
- 3대 벤치마크: **학습시간 / PSNR / floater 수**
- 학습 속도를 위한 선행연구 조사: EDGS(dense correspondence init), Instant-GI(2D GS→3D lift), depth-mono — "좋은 init이 곧 속도이자 품질"
- [시각자료] 3대 벤치마크 다트보드, EDGS/Instant-GI 논문 썸네일
- [발표 노트] 여기서부터 "init 전쟁" 시작을 예고

### 슬라이드 17 — 질문: 다른 방에서도 되는가? (exp43 설계)

- [발표 노트] "앵커 커버리지만 고치면 다른 방에서도 재현된다" — 교차 장면 관문 통과 선언

## §7. 전자동 파이프라인 완성 (슬라이드 22–24) — *v1 유지, 피드백 대기*

### 슬라이드 22 — 라벨 없는 앵커 자가진단 (마지막 퍼즐)
- 새 장면마다 "SLAM 앵커 vs depth 앵커 중 뭘 믿나"를 라벨 없이 판정하는 **2규칙**:
  - 규칙1: SLAM 자기 NN p50 < 0.05m → **SLAM 앵커** (1253 0.042 / rot 0.043 — AUC 0.97+)
  - 규칙2: depth 교차프레임 불일치 p50 < 0.04 → **depth 앵커** (305: 0.032 — AUC 0.886)
  - 규칙3: 둘 다 실패 → **문제 클래스 경고 + carve OFF** (12F: 실제로 carve가 -1dB 손해였음을 학습으로 확인)
- 4/4 장면에서 실측 최선 앵커와 일치
- [시각자료] 진단 플로차트(3분기), 4장면 지표×AUC 매트릭스
- [발표 노트] "이 진단기 덕에 새 장면 파이프라인이 사람 개입 없이 닫혔다"

### 슬라이드 23 — 보너스: SLAM 포인트 없이도 floater를 잡는다
- 질문(사용자): "SLAM 포인트가 아예 없으면?"
- 답: depth-pro 절대 깊이(공장 캘리브레이션으로 focal 검증) + **pose-기하 스케일 자가 보정**(스테레오+IMU 덕에 pose가 미터 단위 → 교차 프레임 일관성으로 스케일 유일 결정)
- 12F 성적: raw 0.855 → 자가 보정 **0.893** (SLAM 보정 상한 0.908의 98%) — **지도 포인트 0개로 성립**
- 단, 학습 통합(vr 채널)은 무효과 → **"탐지≠제거" 간극**: underfit 장면에선 이미지가 먼지를 요구해 어떤 압력도 RGB gradient를 못 이김 → vr의 용도는 오프라인 청소·pseudo-label·탐지
- [시각자료] pose-기하 자가 보정 개념도(두 카메라+depth 삼각형), AUC 사다리(0.855→0.893→0.908)
- [발표 노트] floater의 근본 해법은 압력이 아니라 재구성 품질이라는 통찰로 연결

### 슬라이드 24 — 최종 산출물: 진단-분기 전자동 파이프라인
```
새 장면 VRS
  → SLAM (OpenMAVIS)
  → [자가진단 2규칙]
       ├─ SLAM 양호   → SLAM-앵커 carve      (1253·rot에서 검증)
       ├─ SLAM 희소    → depth-앵커 carve     (305에서 재현 확정, depth 33초)
       └─ 둘 다 불량   → carve OFF + baseline  (12F에서 -1dB 증거)
  → (같은 방 재방문 시) pseudo-label 자동 청소 (정밀도 100%)
```
- 3대 벤치마크 최종: 품질 **+0.9dB** / 먼지 **-83~88%** / 속도 **8분**(fast) — 사람 라벨링 0회로 새 장면 적용 가능
- [시각자료] 위 파이프라인 다이어그램(발표의 마지막 그림), 벤치마크 3종 최종 스코어카드
- [발표 노트] "토요일엔 손실 하나였고, 월요일엔 파이프라인이 됐다"

## §8. 마무리 (슬라이드 25)

### 슬라이드 25 — 결론과 남은 과제
- 확정된 것 5줄:
  1. Carve Loss: 빈 공간 증거 기반 score(AUC 0.98) + 4요소 손실 — PSNR 무손실 먼지 -86%
  2. train PSNR은 floater 지표로 무효(잔차 흡수) → region GT + held-out + SSIM/LPIPS 체계
  3. 교차 장면 재현: 앵커 품질이 단일 지배 변수 — depth-anchor 처방으로 305 재현(재현 런 포함)
  4. 라벨 없는 자가진단 2규칙 + 진단-분기 파이프라인 (4/4 장면)
  5. 연구 위생: 먼지 지표 분산 ×10 → 재현 런 의무화
