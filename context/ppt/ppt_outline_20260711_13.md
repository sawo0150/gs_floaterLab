## §1. 문제의식과 기존 접근 (슬라이드 4–6)

### 슬라이드 4 — 문제의식: 왜 3D 손실이 필요하다고 생각했나 (+ Plateau Loss 소개)
- **Photometric loss의 구조적 사각지대**:
  - photometric loss는 **렌더된 픽셀에 기여하는 gaussian에만 gradient를 전달** → 카메라에서 안 보이거나 드물게 보이는 gaussian은 신호 자체가 없음
  - ray 분포가 **Z 높이별·위치별로 극단적으로 불균등** (많이 훑는 곳 vs 거의 안 훑는 곳)
  - depth loss도 3차원 정보를 주긴 하지만 **역시 ray 기반** — 같은 사각지대 공유
  - → **3차원 공간에 균등하게 신호를 주는 손실이 있어야 기하를 골고루 잡을 수 있다**는 가설
- **최적화 관점의 문제의식**:
  - floater는 **local minima에 갇히는 경향** + 일반 신경망 파라미터와 달리 **gaussian 하나의 차원은 매우 작음** → 저차원에서는 saddle point보다 **local minima가 지배적** → photometric gradient만으로는 탈출이 어려움
  - → **loss landscape를 내려가기 쉽게 smoothing할 필요**
- **⇒ 결론: 3D로 손실을 구현해 (1) 공간 균등 gradient, (2) smoothing으로 local minima 탈출 경로를 만들자**
- **Plateau Loss = 이 문제의식의 1차 구현**:
  - 각 gaussian에 대해 가장 가까운 anchor(SLAM/depth 표면점)까지의 정규화 거리 D를 정의
  - 표면 근처 반경 tau 안쪽은 **평탄(plateau, gradient 0)** — 표면 점을 건드리지 않음
  - plateau 바깥은 D에 비례한 페널티 → 허공 점을 표면 쪽으로 견인
  - 변형들: 구형/타원형(ellipsoidal) plateau, 적응형 tau(anchor 밀도 따라 확대), DepthPro anchor 결합
- [시각자료] ① ray 커버리지 히트맵(높이별 ray 밀도 불균등 — `check_plateau_vs_raydensity.py` 산출물 재활용) ② 1D loss landscape 만화: 뾰족한 local minima vs smoothing 후 ③ plateau 함수 모양 그래프(평탄 구간 + 선형 증가)
- [발표 노트] "이 문제의식 자체는 지금도 유효하다 — 문제는 구현이었다"로 다음 슬라이드 예고

## §2. Carve Loss: 설계 (슬라이드 7–8)

### 슬라이드 7 — Carve Loss의 핵심 아이디어: 빈 공간의 증거
- 직관: **카메라가 SLAM 점을 봤다면, 그 사이 공간은 비어 있어야 한다** (ray casting → free-space carving) — ray가 안 닿는 곳이 아니라 **ray가 '통과했다'는 적극적 증거**를 쓰므로 photometric 사각지대와 무관하게 3D 전역에 정의됨 (슬라이드 4 문제의식의 2차 구현)
- Field 구성 (학습 시작 시 1회, CPU):
  - 카메라→SLAM점 ray를 6cm 간격으로 행진시키며 voxel(10cm)에 **transit(통과) 증거** 적립, 종점 ±20cm에 **terminal(표면) 증거** 적립
  - ρ(x) = transit / (transit + 3·terminal) — "이 voxel이 빈 공간일 증거비" (3×3×3 smoothing — 문제의식의 'smoothing' 요구 반영)
- Score 구성: **w(x) = ρ(x) · min(d5_SLAM/0.25m, 1)** (증거비 × anchor 거리) → **champion score sᵢ = w(xᵢ)·(1−maxop_5cm)** (주변 5cm에 불투명 이웃이 있으면 보호)
- 수동 라벨 판별력: **AUC 0.974~0.98** (plateau 0.511 대비) / 학습 중 갱신 0.38초, 추가 입력 불필요(SLAM+카메라)
- [시각자료] ray casting 개념도(카메라→점, 통과 voxel 초록/표면 voxel 빨강), score 히트맵 단면(방 수평 슬라이스), AUC 비교 막대
- [발표 노트] 기술 핵심 1페이지 — 시간 들여 설명. "plateau가 버리던 raw 거리 신호(0.93)를 증거비와 결합해 0.98로"

### 슬라이드 8 — 손실로 만들기: 4개 구성요소 (수식 상세)
- **① Soft opacity 압력** — score가 높은(빈 공간에 있는) 점의 opacity를 지그시 누름:
  ```
  L_soft = λ_soft · mean_{i: sᵢ > s_min} ( sᵢ · σ(oᵢ) )        [sᵢ는 detach — gradient는 opacity로만]
  champion: λ_soft = 0.02, s_min = 0.5 (softlite)
  ```
  자가 교정적: 진짜 표면이 잘못 눌리면 RGB gradient가 저항해 되살림. λ0.05는 -0.3dB 추가 비용 → 0.02가 스위트스팟
- **② 예산 상한부 Budget Prune** — 시각 기여량으로 부수 피해를 회계 관리하며 삭제:
  ```
  contribᵢ = oᵢ · (s₂·s₃)ᵢ · visᵢ            [불투명도 × 단면적(중간·최대축) × 관측수]
  score 내림차순 정렬 → 누적 contrib ≤ budget 까지 top-K 삭제 (단 sᵢ > 0.3만)
  champion: cycle당 0.05%, 총 상한 0.75% (1k iter마다, 7k 이후)
  ```
  원안(무예산)은 표면 기여 3.83% 손실로 폐기 — 예산제로 0.39% 손실/구멍 0
- **③ Birth Gate** — densify 직후 신생아 검문:
  ```
  kill = (birth_step == 지금 iter) & (sᵢ > 0.95)     [허공 출생 91% 차단 — split의 29.5%가 허공 출생]
  ```
- **④ Carve-potential Force** — 3D 위치를 직접 견인하는 항 (문제의식의 '3D 균등 gradient' 직접 구현):
  ```
  attractor = {terminal 증거 > 1 voxel} ∪ {가시 gaussian(op>0.3) 점유 voxel}
  φ(x) = EDT(¬attractor) · 0.1m                      [빈 공간→가장 가까운 표면까지 거리장]
  L_force = λ_force · mean_{i: sᵢ > 0.5} ( sᵢ · φ(xᵢ) )   [φ는 trilinear 보간 — gradient가 xyz로]
  champion: λ_force = 0.05
  ```
  - **Plateau에서의 진화 — attractor(anchor)를 늘렸다**: plateau의 견인 목표는 **희소 SLAM anchor 점뿐**(7천 개) → anchor가 없는 천장·텍스처리스 표면에서는 점을 엉뚱한 먼 anchor로 끌어 **RGB 손실이 컸음**. carve force는 attractor를 **terminal 증거 voxel ∪ 이미 재구성된 가시 gaussian 점유**로 확장 — '표면'의 정의를 "anchor 점"에서 "표면이라는 증거가 있는 모든 곳"으로 넓혀, 어디서든 가장 가까운 진짜 표면으로 짧게 견인
  - gradient 프로브 근거: floater는 강한 핀이 아니라 **진동 평형**(gradient coherence 0.257) 위에 앉아 있음 → 약한 '일관' 편향이면 이탈 가능
- 4요소의 분업: gate(출생 차단) → soft(성장 억제) → force(위치 견인) → prune(정리). 전부 같은 score 하나에서 파생
- [시각자료] 각 수식 박스 4개 + 타임라인(iter 0—1k—7k—30k 위에 각 요소 활성 구간), φ 거리장 단면 히트맵, plateau attractor(점 몇 개) vs carve attractor(면 전체) 비교 그림
- [발표 노트] "누르고(1), 자르고(2), 안 낳게 하고(3), 끌어당긴다(4)" 한 줄 요약. detach 위치(score는 상수 취급) 질문 대비

## §4. 지표의 재정비 (슬라이드 12–13)

### 슬라이드 12 — 평가 체계 재구축: train PSNR의 사각지대와 3종 지표
- 렌더 A/B 중 확인: **수동으로 floater를 지우기만 해도 train PSNR -3.7dB** — floater가 train 이미지의 잔차(글레어·노출)를 흡수해 PSNR을 벌어줌 → train PSNR은 floater 평가에 부적합 (슬라이드 6의 "RGB gradient 2.23×"의 귀결)
- 재구축한 3종 체계:
  - ① **Region GT 지표** (슬라이드 5의 사면체 영역 재사용): 어떤 학습 결과든 "영역 안 점/가시 점 수"로 채점
  - ② **held-out 평가**: every-8th는 보간이라 잔차 흡수가 test에서도 이김(-0.46dB) → **연속 세그먼트 hold-out**으로 교체, 두 split 일치로 비용 확정
  - ③ **SSIM/LPIPS**: 챔피언 vs baseline **사실상 동률**(SSIM 0.9152 vs 0.9167, LPIPS 0.3668 vs 0.3633)
- **Pareto 지도(exp42)** — 먼지↔PSNR 트레이드는 연속 조절 가능:

| 운영점 | test PSNR Δ | region_n | 가시 |
|---|---:|---:|---:|
| baseline | — | ~3,700 | ~200 |
| prune+gate | -0.09 | 1,795 | 217 |
| +force | -0.21 | 1,136 | 149 |
| softlite+force (챔피언) | **-0.44** | **~460** | **~25** |

- 최종 그림: **챔피언 = 지각 품질 동등 + 가시 먼지 -90%+ + PSNR -0.44dB(비지각적 잔차 맞춤의 상실)** — λ·budget으로 운영점 조절 가능
- [시각자료] Pareto 곡선(x=먼지, y=test PSNR, 운영점 4개 표시), SSIM/LPIPS 동률 표, region 영역 3D 오버레이(슬라이드 5 재등장)
- [발표 노트] 담백하게: "지표를 고쳐야 공정한 실험이 가능했다" — 과장 없이 체계 소개 중심

## §5. Init 전쟁 (슬라이드 14–17)

### 슬라이드 14 — Init 전쟁의 등장인물: 방법론 소개
> 처음 보는 동료를 위한 용어 정리 — 이후 표에 나오는 이름들의 정의

- **depth-mono (Depth Pro)**: 이미지 1장 → 픽셀별 절대 깊이(단안 depth 추정, ~1초/장). 스케일 오차가 있어 **SLAM 점으로 보정 후 사용**(알고리즘은 슬라이드 21)
- **depth-lift init**: 보정된 depth로 이미지 픽셀을 3D로 역투영해 dense 점군 생성 — 텍스처 없는 흰 벽·천장도 커버 (SLAM/matching의 사각지대 보완)
- **PPM (Position Probability Map, Instant-GI 사상)**: "점을 어디에 뿌릴까"를 이미지 내용으로 결정 — 텍스처 복잡한 곳에 촘촘히, 평탄한 벽엔 성기게. 확률맵 비례 샘플링 후 depth-lift. 두 구현: **수제 Sobel**(경계 강도) vs **Instant-GI 사전학습 ConvNeXt**(가중치 다운로드·추론)
- **RoMA (EDGS 사상)**: 두 이미지 사이 **dense correspondence**(픽셀 대응 수만 쌍)를 찾는 매칭 네트워크 → SLAM pose로 **삼각측량** → 재투영 오차 <2px만 통과 = **기하학적으로 검증된** 3D 점 (단, 텍스처 있는 곳만 가능)
- **carve 사전 필터**: init 점군에 carve score를 적용해 빈 공간 점을 학습 전에 제거 (w≤0.5/0.9 두 강도)
- **스냅(snap-to-surface)**: 필터로 버릴 빈 공간 점을 ray 방향 표면 증거 깊이로 이동시켜 재활용
- **투영 착색**: init 점을 이미지에 투영해 색을 미리 입힘 (회색 시작 대비 색 최적화 시간 절약)
- [시각자료] 방법론별 아이콘+1줄 카드 7장, RoMA 매칭 시각화(대응선), PPM 확률맵 히트맵(Sobel vs ConvNeXt 나란히), depth-lift 개념도
- [발표 노트] "init 재료 3종(depth/PPM/RoMA) × 후처리 3종(필터/스냅/착색)의 조합 탐색"이라고 구조화

### 슬라이드 20 — 진단: 원인은 'ORB 지도의 표면 커버리지'

### 슬라이드 21 — 도구: 단안 depth를 미터 단위로 — SLAM 기반 보정 알고리즘
> init(PPM lift)과 처방(depth-anchor) 공통의 기반 기술 — 별도 정리

- 문제: depth-pro의 깊이는 픽셀별 상대 구조는 정확하나 **장면별·프레임별 스케일 오차** 존재 (12F 실측: raw 스케일 계수 1.274, 프레임 간 IQR [0.87, 1.52])
- **프레임별 Huber 보정** (사용 중인 알고리즘):
  1. 그 프레임에 **보이는 SLAM 점들을 투영** → 해당 픽셀의 depth-pro 값 D 샘플
  2. 강건 선형 회귀(HuberRegressor, ε=1.35): **z_SLAM ≈ a·D + b** — outlier(SLAM 오점·occlusion)에 강함
  3. 보정 깊이 = max(a·D + b, 0.15m) → 전 픽셀에 적용
- 보정 깊이의 용도 3종: ① PPM/depth-lift init의 3D 역투영 ② depth-anchor 점군 생성(grid 역투영→5cm dedupe) ③ depth-violation 검사(슬라이드 25)
- 검증: 공장 캘리브레이션 확인 — 파이프라인이 VRS device calibration으로 RGB를 **정확한 pinhole(f=500)로 정류**하므로 f_px 입력은 옳음 → 잔여 오차는 depth-pro 자체의 장면 편향(fog에서 큼)
- 한계와 대안: SLAM 점이 아예 없으면? → **pose-기하 자가 보정**(슬라이드 25): 스테레오+IMU 덕에 pose 이동량이 미터 단위 → 두 프레임 depth에 같은 s를 곱해도 카메라 이동은 고정이므로 교차 프레임 일관성이 s를 유일 결정
- [시각자료] 보정 산점도(D vs z_SLAM + Huber 직선), 보정 전/후 depth 맵 나란히, 프레임별 스케일 계수 분포(안정 장면 vs fog 장면)
- [발표 노트] "SLAM이 '자'가 되고 depth가 '살'이 된다" — 희소한 정확함 × 조밀한 부정확함의 결합

### 슬라이드 25 — 확장: SLAM 포인트가 아예 없어도 되는가
- 동기: 실시간 경로에서 지도 품질 의존을 더 줄일 수 있나 (pose는 스테레오+IMU로 항상 미터 단위)
- **depth-violation ratio (vr) — 눈높이 설명**:
  - violation = '위반'. depth-mono가 "이 방향 3m에 벽이 있다"고 말했는데 어떤 gaussian이 그 **벽보다 앞쪽(예 1.5m) 허공**에 떠 있으면 → 카메라와 벽 사이 빈 공간에 있으니 floater 의심
  - gaussian 하나에 대해: 여러 depth 프레임에 투영 → 각 프레임에서 "**측정된 표면보다 카메라 쪽에 떠 있나?**"(위반 1표) → **vr = 위반한 프레임 수 / 보인 프레임 수** ∈ [0,1]
  - vr≈1 = 보이는 거의 모든 뷰에서 벽보다 앞 = 강한 floater 신호 / vr≈0 = 늘 표면에 붙음 = 정상
  - carve의 원래 신호(ray 통과 증거 ρ)는 **3D voxel 격자** 기반이라 fog에서 흐려지는데, vr은 **이미지 공간**에서 직접 재므로 fog에서도 잘 잡힘
- 12F(최난도 장면) 실측 사다리:

| 신호 | SLAM 점 사용 | AUC |
|---|---|---:|
| vr (raw depth, 스케일 무보정) | 불필요 | 0.855 |
| **vr + pose-기하 자가 보정** (s=1.425 추정, SLAM Huber는 1.274) | **불필요 (pose만)** | **0.893** |
| vr + SLAM Huber 보정 (상한) | 필요 | **0.908 — 12F 신기록** |

- 그러나 **학습 통합은 무효과** (12F 페어 실험):

| 12F 30k | PSNR | region_n | 가시 |
|---|---:|---:|---:|
| baseline | **32.034** | 2,703 | **120** |
| carve (ctrl) | 31.044 | 1,310 | 221 |
| carve + vr 채널 (score=max(s,vr)) | 30.925 | 1,204 | 193 |

- **"탐지≠제거" 간극**: underfit 장면에선 이미지가 먼지를 요구 → 어떤 score 압력도 RGB gradient를 못 이김 (rot 질량 증폭과 동일 역학). **floater의 근본 해법은 압력이 아니라 재구성 품질**
- vr의 채택 용도: 오프라인 청소·pseudo-label·SLAM-프리 탐지 (학습 손실 아님)
- [시각자료] pose-기하 자가 보정 개념도(두 카메라+depth, 스케일 s의 유일 결정), AUC 사다리 막대, 페어 표
- [발표 노트] 부정적 결과(학습 통합 무효과)를 통찰("탐지≠제거")로 전환하는 슬라이드

### 슬라이드 26 — 최종 산출물: 진단-분기 전자동 파이프라인

## §8. 심화 분석과 마무리 (슬라이드 27–29)

### 슬라이드 27 — 심화 분석: "좋은 init을 줘도 underfit이 남는가?"
- 질문: floater의 근본 원인이 underfit이라면, init을 잘 주면 underfit이 사라져 floater도 사라져야 하지 않나?
- **답: underfit은 원인이 두 종류이고, init이 통하는 쪽과 안 통하는 쪽이 갈린다.**

| underfit 유형 | 원인 | 좋은 init이 돕나 | 근거 |
|---|---|---|---|
| **(A) 기하 과소구속** — rot(회전 궤적) | 같은 표면 보는 각도 부족(시차 부족) → 이미지로 기하 삼각측량 불가 | **예 (+1.37dB)** | 실측: rot hybrid init → PSNR 30.689→32.056 |
| **(B) 지도(supervision) 열화** — 12F(fog·어두움) | 맞출 이미지 자체가 흐림·저텍스처 → 잔차가 기하로 안 줄어듦 | **아마 아니오** | 미검증 (예측) |

- **(A)의 논리**: init이 이미지로는 못 잡는 기하를 대신 공급 → 잔차↓ → 먼지 수요↓. 실제로 rot은 init으로 +1.37dB. 단 시차 자체가 부족해 잔여 수요는 남음(완전 해소는 아님)
- **(B)의 논리**: floater는 '이미지 잔차 흡수 기생충'인데, fog 이미지의 잔차는 **기하를 잘 잡아도 안 사라지는 종류**(안개=시점의존 볼류메트릭, 어떤 gaussian 배치로도 정확히 못 맞춤) → init을 바꿔도 잔차 수요가 남아 먼지 수요도 남음. 이것이 슬라이드 25 "탐지≠제거" 간극의 뿌리
- **정직한 구멍**: (B)에 좋은 init을 실제로 넣어본 적 없음 — 12F 학습은 전부 raw SLAM init 위에서, depth 앵커는 carve field 재료로만 사용. "12F에 hybrid/depth-lift init을 주면 천장(~32)이 오르나?"는 **예측만 있고 미검증**
- [시각자료] (A)/(B) 2분할 개념도(A: 카메라 각도 부족 그림, B: fog 이미지가 잔차를 남기는 그림), rot init 전후 PSNR 막대(30.69→32.06), "미검증" 표시된 12F 칸
- [발표 노트] "floater = underfit의 증상. underfit의 원인이 시작점(init)이면 고쳐지고, 목표 이미지면 안 고쳐진다"

### 슬라이드 28 — 앞으로 해볼 것 (검증 대기 가설과 다음 실험)
- **① (B)유형 구멍 메우기 — 즉시 실행 가능**: 12F에 hybrid/depth-lift init + carve 학습 1회 (~15분, depth 앵커·보정 depth 이미 준비됨). 예측: 천장 거의 안 오름 → 맞으면 "지도 열화형은 init 무효" 확정, 틀리면 (B) 재해석
- **② 회전 궤적 축 (rot underfit 정면 돌파)**: 시차 보장 쌍 선택(min-baseline)으로 삼각측량 개선 — 1차 시도는 미해결(hybrid ×10 먼지). depth 앵커도 불량(회전 궤적에서 depth 보정 붕괴, AUC 0.85). 남은 카드: 다중 뷰 색 일관성, 렌더-GT 잔차 신호
- **③ 표면 부착 blob (305 라벨의 30%)**: free-space 신호(ρ·vr) 밖의 새 문제 클래스 — 표면에 붙은 불량 blob은 "빈 공간" 논리로 못 잡음. 별도 신호(멀티뷰 색 불일치 등) 필요
- **④ "탐지≠제거" 간극 뚫기 (근본 과제)**: vr은 탐지 최강(0.908)인데 학습 압력은 무효 → 압력이 아닌 경로 필요. 후보: 노출/글레어 명시 모델링(잔차의 근원 제거 — 45a 재설계), init·시차로 잔차 수요 자체를 없애기
- **⑤ 파이프라인 원커맨드화 + 새 장면 검증**: 자가진단→분기→학습을 `run_scene_pipeline.sh`에 통합, 새 장면 1개로 자가진단 2규칙의 5번째 검증(현재 n=4)
- [시각자료] 5개 다음 실험 카드(각: 가설 / 실행 준비도 / 예상 결과), 우선순위 화살표
- [발표 노트] "①은 오늘 돌릴 수 있고, ④가 근본 과제" — 청중에게 열린 질문으로 남김

### 슬라이드 29 — 결론과 핵심 메시지
- 확정된 것 5줄:
